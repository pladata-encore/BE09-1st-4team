## 7. MSA 설계

### 7.1 시스템 구조

Starfox 키오스크 시스템은 MSA(Microservices Architecture) 구조로 구성되며, 각 서비스는 독립적으로 운영됩니다. REST API 기반으로 서비스 간 통신이 이루어집니다.

### 7.2 서비스 구성

| 서비스명 | 주요 기능 | 데이터베이스 |
| :--- | :--- | :--- |
| User Service | 메뉴 탐색, 상품 선택, 장바구니, 주문 결제 | user_db |
| Admin Service | 상품 등록/수정/삭제, 주문 상태 관리 | admin_db |
| Order Service | 주문 생성, 결제 처리, 주문 상태 업데이트 | order_db |
| Product Service | 상품 및 카테고리 조회 및 관리 | product_db |

### 7.3 데이터 흐름

#### 사용자 주문 흐름
1. 사용자는 카테고리 및 상품을 탐색  
2. 옵션 및 수량을 설정 후 장바구니에 담음  
3. 결제 후 주문 요청 → Order Service로 전달  
4. 주문 생성 및 상태 업데이트 진행

#### 관리자 주문 처리
1. 대기 중인 주문 조회  
2. 제조 완료 처리  
3. 주문 상태 변경 → 사용자 측 반영

## 8. StoryBoard

[User]  
메인 화면  
→ 카테고리 선택 (음료/푸드/디저트 등)  
→ 메뉴 목록 선택  
→ 상품 상세 화면 (옵션 선택, 수량/사이즈 선택)  
→ 장바구니 담기  
→ 장바구니 확인 및 수정  
→ 결제하기  
→ 주문 완료 및 대기 목록 등록

[Admin]  
관리자 메인 화면  
→ 상품 관리 / 옵션 관리 / 주문 관리 메뉴 선택  
→ 상품 목록 조회 / 추가 / 수정 / 삭제  
→ 옵션 목록 조회 / 추가 / 수정 / 삭제  
→ 주문 목록 조회 및 제조 완료 처리

## 12. User 영역 주요 기능

### MenuView 클래스 기능

- 카테고리 선택 (`inputCategory`)  
- 메뉴 선택 (`inputMenu`)  
- 옵션 및 수량/사이즈 선택 (`selectOptions`)  
- 장바구니 출력 (`printCart`)  
- 장바구니 메뉴 조작 (`inputCartMenu`, `inputModifyMenu`)  
- 주문 내역 출력 (`printOrder`)

## 13. Admin 영역 주요 기능
